### 타이포그래피 (Pretendard 추정)
- 페이지 타이틀: 24px / 700
- 섹션 타이틀: 18px / 600 (예: `발주계약요청 목록`)
- 카운트 강조: 16px / 700, 오렌지 (`19건`, `24건`)
- 테이블 헤더: 14px / 500, 회색
- 테이블 셀: 14px / 400
- 라벨(필터): 13px / 500
- 사이드바 메뉴: 15px / 500

### Radius / Spacing
- 카드 radius: 12px
- 입력 필드 radius: 8px
- 버튼 radius: 8px
- 배지 radius: 9999px (pill)
- 카드 내부 padding: 24px
- 필터 영역 padding: 24px
- 테이블 row 높이: 약 48px

## 2. 공통 레이아웃

### 2.1 Top Header (전 화면 공통)
```
[로고] [☰]                                     [업무시스템 이동 ▼]   [H 사업본부 개발팀 홍길동]   로그아웃
```
- 좌: KEPCO ES 로고 + 햄버거 아이콘
- 우: 시스템 이동 드롭다운 / 사용자 배지(H 아바타 + 부서·이름) / 로그아웃 텍스트 링크
- 높이: 약 64px, 하단에 옅은 보더

### 2.2 Left Sidebar (전 화면 공통)
- 폭: 약 240px
- 상단: 메뉴 검색 인풋 (`메뉴를 검색하세요.` placeholder + 돋보기 아이콘)
- 메뉴 항목 (아이콘 + 텍스트):
  1. 견적
  2. 발주
  3. 입찰
  4. 평가
  5. 낙찰
  6. 계약
  7. 마이페이지
  8. 시스템 관리
- **액티브 상태**: 오렌지 배경(필) + 흰색 아이콘·텍스트, 좌우 마진을 둔 둥근 버튼 형태
- 비액티브: 진한 회색 텍스트, 호버 시 옅은 배경

### 2.3 페이지 헤더 (콘텐츠 영역 상단)
```
< 페이지명
```
- 좌측에 `<` 뒤로 가기 아이콘 + 페이지 타이틀 (24px / 700)
- 마진: 위 32px, 아래 24px

## 3. 화면별 스펙

### 3.1 발주계약 관리 (`/발주` 또는 `/order`)

**구조**
1. 페이지 헤더: `< 발주계약 관리`
2. 필터 카드 (베이지 배경)
   - 4-컬럼 그리드 + 우측 버튼 영역
   - 필드:
     - `기간 *` — 날짜 범위 인풋 (캘린더 아이콘) — 예시값 `2026.05.01~2026.05.20`
     - `진행상태` — 셀렉트 (`전체` 기본)
     - `체크리스트` — 셀렉트 (빈 상태)
     - `키워드` — 텍스트 인풋
   - 우측 액션: `초기화` (ghost) · `검색` (브라운 fill)
3. 리스트 카드
   - 헤더: 좌 `발주계약요청 목록 19건` / 우 `체크리스트 작성`(브라운) + `📊 다운로드`(흰 + Excel 초록 아이콘)
   - 테이블 컬럼: ☐ / 요청번호 / 계약명 / 구분 / 요청일 / 수주계약번호 / 수주계약명 / 체크리스트 / 진행상태
   - **체크리스트 컬럼**: `작성중` (블루 pill) / `완료` (오렌지 pill)
   - **진행상태 컬럼**: 텍스트 only — 계획(블루) · 입찰공고(오렌지) · 업체평가(그린) · 개찰(틸)
   - 행 12개 표시 (시안 기준)

**기존 컴포넌트 매핑 후보**
- `PageHeader.tsx` ← 페이지 헤더
- `SearchForm.tsx` ← 필터 카드
- `DataTable.tsx` ← 리스트 테이블
- `StatusBadge.tsx` ← 작성중/완료 pill

### 3.2 사용자관리 (`/시스템관리/사용자관리`)

**구조**
1. 페이지 헤더: `< 사용자관리`
2. 탭 (`그룹관리` 액티브 / `사용자관리` / `권한설정`)
   - 액티브: 진한 텍스트 + 하단 오렌지 굵은 언더라인
   - 비액티브: 회색 텍스트, 언더라인 없음
3. 2-컬럼 영역
   - **좌측 카드 (`그룹목록`)** — 폭 약 320px
     - 헤더: `그룹목록` + 우측 `+ 그룹추가` 텍스트 링크
     - 그룹 카드 리스트 (3개):
       - `관리자` (액티브, 오렌지 배경) + `전체` 배지
         - 서브: `시스템관리자/총괄관리자 2명`
       - `사업담당자` + `일부` 배지(시안)
         - 서브: `운영자 1명`
       - `계약담당자` + `일부` 배지
         - 서브: `운영자 1명`
   - **우측 카드 (`플랜트사업소 - 접근권한`)** — 가변 폭
     - 헤더: 타이틀 + 우측 `저장`(브라운) + `그룹삭제`(흰 outline)
     - 본문: `메뉴 (페이지) 접근권한` 라벨 아래 4-컬럼 체크박스 그리드
       - 견적요청 조회 / 발주계약요청 조회 / 발주계약계획 등록 / 입찰참여 신청
       - 견적요청 등록 / 발주계약요청 등록 / 입찰계획 조회 / 입찰참여 자가심사
       - 견적요청 조회 / 발주계약요청 접수 / 입찰계획 등록 / 입찰참여 투찰
       - 견적서 등록 / 발주계약계획 조회 / 입찰공고 조회 /
4. 하단 카드 (`소속 사용자`)
   - 헤더: 타이틀 + 우측 `사용자추가` (브라운)
   - 테이블 컬럼: 이름 / 아이디 / 역할 / 상태 / 마지막접속 / 관리
   - 역할 배지: `시스템관리자`(블루) · `총괄관리자`(핑크)
   - 상태: `활성` (오렌지 텍스트)
   - 마지막접속: `● 접속중` (그린)
   - 관리: `✏ 편집`(오렌지 텍스트+아이콘) · `× 삭제`(회색)

**기존 컴포넌트 매핑 후보**
- `Tabs.tsx` ← 그룹관리/사용자관리/권한설정
- `DataTable.tsx` ← 소속 사용자 테이블
- `StatusBadge.tsx` ← 역할·상태 배지
- 신규 필요: `GroupListCard` (좌측 그룹 카드 리스트), `PermissionGrid` (체크박스 그리드)

### 3.3 입찰공고 (`/입찰/입찰공고`)

**구조**
1. 페이지 헤더: `< 입찰공고`
2. 필터 카드 (3-컬럼 + 버튼)
   - `기간 *` — 날짜 범위
   - `계약방법` — 셀렉트 (`전체`)
   - `키워드` — 텍스트
   - 우측: `초기화` · `검색`
3. 리스트 카드
   - 헤더: 좌 `입찰 목록 24건` / 우 `📊 다운로드` 만 (체크리스트 작성 버튼 없음)
   - 테이블 컬럼: ☐ / 입찰공고번호 / 입찰건명 / 구분 / 공고일자 / 계약방법 / 심사방법 / 담당자 / 마감일시
   - 모든 셀 plain 텍스트 (배지·컬러 없음)
   - 행 12개 표시 (시안 기준)

**기존 컴포넌트 매핑 후보**
- `PageHeader.tsx` / `SearchForm.tsx` / `DataTable.tsx` 그대로 재사용

## 4. 화면 간 패턴 비교

| 패턴 | 발주계약 관리 | 사용자관리(그룹관리) | 입찰공고 |
|---|---|---|---|
| 필터 카드 | ✅ 4컬럼 | ❌ | ✅ 3컬럼 |
| 탭 | ❌ | ✅ 3개 탭 | ❌ |
| 메인 테이블 | ✅ 상태배지 포함 | ✅ 하단 카드 (배지 다양) | ✅ plain |
| 다운로드 버튼 | ✅ | ❌ | ✅ |
| 보조 액션 | 체크리스트 작성 | 사용자추가 / 저장·삭제 | — |

→ **공통 셸**: Header + Sidebar + PageHeader는 모든 화면이 동일.
→ **반복 패턴**: 필터 카드 + 리스트 카드 (2개 화면) — 컴포넌트 1개로 추출 가능.
