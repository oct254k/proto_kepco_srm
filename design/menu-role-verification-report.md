## 3. 문서 대비 판정

| 역할 | 기준 문서 기대 구조 | 실제 구현 판정 | 비고 |
|---|---|---|---|
| 사업담당자(B) | `참여업체평가`가 독립 메뉴, `입찰`은 조회 메뉴 중심 | 부분일치 | `입찰심사`가 `입찰` 하위로 흡수되어 상위 메뉴 수가 6→5로 축소됨 |
| 협력업체(V) | `입찰`, `입찰참여관리`, `계약`이 분리 | 부분일치 | `입찰`+`입찰참여관리` → `입찰참여`, `계약` → `계약·보증`으로 통합됨 |
| 계약담당자(C) | `견적`, `발주`, `입찰`, `계약`이 각각 독립 | 부분일치 | `견적`이 `발주관리` 하위로 편입되고, 상위 명칭이 `발주관리/입찰관리/계약관리`로 운영형 라벨로 바뀜 |
| 관리자(A) | `기준정보 관리`, `시스템 환경설정` 표기 | 부분일치 | 실제 메뉴명은 `기준정보(품목)`, `시스템환경`으로 축약되어 있음 |

## 4. 역할별 접근 검증 결과

| 검증 항목 | 결과 | 근거 |
|---|---|---|
| 메뉴 정의와 기준선 일치 | PASS | `src/lib/menu.ts`를 `node scripts/verify-menu-structure.mjs`로 검증 |
| 메뉴 href별 실제 페이지 존재 | PASS | `src/app/**/page.tsx` 26개 라우트 자동 대조 |
| 기본 랜딩 경로 일치 | PASS | `src/lib/access.ts`의 `DEFAULT_PATHS` 대조 |
| 역할 prefix 접근 제어 일치 | PASS | `/b/`, `/v/`, `/c/`, `/a/` prefix와 메뉴 href 대조 |
| 문서상 상위 메뉴 명칭 완전 일치 | FAIL | 프로토타입은 실제 운영형 메뉴명으로 일부 통합·축약되어 있음 |

## 5. 해석 및 권고

1. **현재 프로토타입의 실제 메뉴 기준선은 구현상 안정적**이다.
   - 메뉴 정의, 기본 진입 경로, 역할별 라우트 연결은 모두 정상이다.
2. **문서와 구현의 차이는 오류라기보다 통합/축약 결정**에 가깝다.
   - 특히 B/V/C 역할은 상위 메뉴를 줄이고 하위 기능을 흡수하는 방향으로 단순화됐다.
3. 이후 리더 문서 산출물에서는 아래 중 하나를 반드시 선택해야 한다.
   - 문서를 현재 프로토타입 기준 메뉴명으로 동기화
   - 또는 프로토타입 메뉴명을 문서 기준으로 되돌려 용어를 일치화
4. 우선순위는 **C 역할 `견적` 독립 여부**, **V 역할 `입찰`/`입찰참여관리` 분리 여부** 확인이다.
   - 두 지점이 역할별 업무 분장 설명에 가장 큰 영향을 준다.
